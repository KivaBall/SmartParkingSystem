# Smart Parking System — інструкція по підключенню заліза для Arduino Mega 2560

Цей документ — це основна roadmap-інструкція для актуальної збірки на **Arduino Mega 2560**. Тут описано **що купити, де це з'єднати, як пов'язані паркомісця з датчиками та LED-маршрутами, і як перевірити що воно живе**.

> **Актуальна цільова плата — Arduino Mega 2560.** Arduino Uno/Nano для нової версії з RFID, LCD, Bluetooth, 4 ультразвуковими датчиками, сервоприводом і 3 адресними LED-стрічками вважаються застарілими: на них не вистачає зручних пінів і запасу пам'яті. Старий Uno-док залишений тільки як архів.

## 1. Що ми взагалі збираємо

Це контролер розумного паркінгу. Він уміє:

- **Ворота на сервоприводі** — відкриваються по RFID-картці або по команді з застосунку.
- **Датчики паркомісць** — 3 ультразвукові сенсори, кожен бачить чи стоїть машина на конкретному місці.
- **Датчик проїзду під воротами** — четвертий ультразвуковий, щоб ворота автоматично закривались після того як машина проїхала.
- **RFID-зчитувач** — підносиш картку, він її розпізнає.
- **Bluetooth HC-05** — застосунок на телефоні підключається і керує всім дистанційно.
- **LCD дисплей 16x2** — показує статус системи (Scan Card, Access Granted, Blocked Card тощо).
- **3 адресні LED-стрічки WS2812B/NeoPixel** — кожна показує маршрут до свого фізичного паркомісця: P1, P2 або P3.

## 4. Карта пінів Arduino

Це найважливіша таблиця. **Кожен дріт має йти саме в той пін, що тут написано** — інакше прошивка не запрацює.

### Цифрові піни

| Пін | Куди йде | Призначення |
|-----|----------|-------------|
| D0 (RX) | — | НЕ використовувати (USB-комунікація) |
| D1 (TX) | — | НЕ використовувати (USB-комунікація) |
| **D2** | HC-05 **TX** | Bluetooth прийом за поточною прошивкою (`SoftwareSerial`) |
| **D3** | HC-05 **RX** *(через дільник напруги!)* | Bluetooth передача за поточною прошивкою (`SoftwareSerial`) |
| **D4** | HC-SR04 #2 **TRIG** | Слот 2 — імпульс |
| **D5** | Сервопривід **сигнал** (помаранчевий/жовтий) | Керування воротами |
| **D6** | HC-SR04 #2 **ECHO** | Слот 2 — відповідь |
| **D7** | HC-SR04 #1 **TRIG** | Слот 1 — імпульс |
| **D8** | HC-SR04 #1 **ECHO** | Слот 1 — відповідь |
| **D9** | MFRC522 **RST** | Скидання RFID |
| **D10** | MFRC522 **SDA/SS** | Вибір RFID по SPI |
| **D22** | DATA LED-стрічки маршруту до **P1** | Підсвітка дороги до паркомісця 1 |
| **D23** | DATA LED-стрічки маршруту до **P2** | Підсвітка дороги до паркомісця 2 |
| **D24** | DATA LED-стрічки маршруту до **P3** | Підсвітка дороги до паркомісця 3 |
| **D51** | MFRC522 **MOSI** | SPI дані → RFID на Mega |
| **D50** | MFRC522 **MISO** | SPI дані ← RFID на Mega |
| **D52** | MFRC522 **SCK** | SPI такт на Mega |

### Аналогові та I2C-піни

| Пін | Куди йде | Призначення |
|-----|----------|-------------|
| **A0** | HC-SR04 #3 **TRIG** | Слот 3 — імпульс |
| **A1** | HC-SR04 #3 **ECHO** | Слот 3 — відповідь |
| **A2** | HC-SR04 #4 **TRIG** | Датчик проїзду — імпульс |
| **A3** | HC-SR04 #4 **ECHO** | Датчик проїзду — відповідь |
| **D20 / SDA** | LCD **SDA** | I2C дані на Mega |
| **D21 / SCL** | LCD **SCL** | I2C такт на Mega |

### Живлення

| Контакт Arduino | Що до нього йде |
|-----------------|-----------------|
| **5V** | HC-05 VCC, всі 4× HC-SR04 VCC, **LCD VCC** *(коли сервопривід живиться окремо)* |
| **3.3V** | MFRC522 VCC ⚠️ **тільки 3.3V, не 5V — інакше згорить!** |
| **GND** (їх кілька — використовуй усі) | Спільна земля для **всіх** модулів і зовнішнього блоку живлення |
| **Vin** | Не чіпати при USB-живленні |

### Кореляція паркомісць, датчиків і LED-маршрутів

Ця таблиця головна для макета. Якщо застосунок вибрав `PARKING ROUTE 1`, Arduino вмикає саме LED-стрічку для маршруту до P1. Датчик цього ж паркомісця окремо показує, чи місце зайняте.

| Паркомісце | Розташування на макеті | Датчик зайнятості | TRIG | ECHO | LED-маршрут | DATA-пін LED | Команда |
|------------|------------------------|-------------------|------|------|-------------|--------------|---------|
| **P1** | Перше фізичне місце, найближче до в'їзду | HC-SR04 #1 | **D7** | **D8** | LED strip #1 | **D22** | `PARKING ROUTE 1` |
| **P2** | Друге фізичне місце, середина маршруту | HC-SR04 #2 | **D4** | **D6** | LED strip #2 | **D23** | `PARKING ROUTE 2` |
| **P3** | Третє фізичне місце, дальнє місце | HC-SR04 #3 | **A0** | **A1** | LED strip #3 | **D24** | `PARKING ROUTE 3` |

Логічні слоти **P4-P6** залишаються в протоколі застосунку на майбутнє, але у поточному фізичному макеті для них немає ні ультразвукових датчиків, ні LED-маршрутів.

Щоб вимкнути всі маршрути, застосунок або Serial Monitor має надіслати:

```text
PARKING ROUTE_CLEAR
```

## 5. Підключення крок за кроком

> 🔌 **Перед кожним підключенням — від'єднай Arduino від USB.** Ніколи не з'єднуй живлення на платі, що ввімкнена.

### Крок 0 — підготовка breadboard

1. Постав велику breadboard поруч з Arduino.
2. На breadboard є дві довгі шини зверху і знизу: червона (+) і синя (–).
3. **Зроби три перемички ще до всього іншого:**
   - Arduino **5V** → червона шина breadboard (це буде "+5V" для всіх 5-вольтових модулів).
   - Arduino **GND** → синя шина breadboard (це буде "GND" для **всього**).
   - Якщо в тебе є зовнішнє живлення 5V — підключи його **мінус** теж до синьої шини. **Землі мають бути спільні!**
4. Якщо плануєш зовнішнє живлення для сервоприводу/датчиків — заведи його **+5V** на **окрему** червону шину (не з'єднуй з 5V Arduino).

---

### Крок 1 — Сервопривід (ворота)

Сервопривід має 3 дроти:
- **Коричневий або чорний** = GND
- **Червоний** = +5V
- **Помаранчевий або жовтий** = сигнал

Підключення:

| Дріт сервоприводу | Куди |
|-------------------|------|
| GND (коричневий) | Синя шина (GND) |
| +5V (червоний) | **Зовнішня червона шина** (краще), або 5V Arduino (якщо нічого більше не плануєш) |
| Сигнал (жовтий) | Arduino **D5** |

> ⚠️ **Чому окреме живлення?** Сервопривід при русі споживає до 600 мА. Arduino через USB видає максимум ~500 мА на всю плату. Якщо живити сервопривід від Arduino, плата буде перезавантажуватись щоразу при відкритті воріт. Якщо є можливість — зразу зроби зовнішнє живлення 5V 2A.

---

### Крок 2 — Bluetooth HC-05 ⚠️ найскладніший крок

Модуль має 6 пінів, але нам потрібні тільки 4: **VCC, GND, TXD, RXD**.

**Проблема:** HC-05 живиться від 5V, але його логіка на 3.3V. Arduino видає сигнал 5V на TX, що може спалити RX HC-05. Тому між **Arduino D3 → HC-05 RX** треба зробити **дільник напруги** з резисторів 1кОм і 2кОм.

**Дільник напруги (просто, на mini-breadboard):**

```
Arduino D3 ──[1 кОм]──┬──[2 кОм]── GND
                           │
                           └─→ HC-05 RX (тут буде ~3.3V)
```

Підключення:

| HC-05 пін | Куди |
|-----------|------|
| VCC | Червона шина (+5V) |
| GND | Синя шина (GND) |
| TXD | Arduino **D2** *(напряму, без дільника)* |
| RXD | Через дільник з Arduino **D3** |
| EN / KEY | Не чіпати |
| STATE | Не чіпати |

> 💡 Можна без дільника — більшість HC-05 виживуть, але це не за стандартом. Для одноразового макета зійде; для серйозної збірки — роби дільник.

> Поточна прошивка використовує `SoftwareSerial btSerial(2, 3);`, тому для Mega у цьому документі Bluetooth теж лишається на D2/D3. Якщо upload поводиться дивно — тимчасово від'єднай HC-05 TX/RX.

### Крок 3 — RFID-модуль MFRC522

**КРИТИЧНО: живлення 3.3V, не 5V!** Якщо подаси 5V — модуль згорить.

| MFRC522 пін | Arduino пін |
|-------------|-------------|
| VCC | **3.3V** ⚠️ |
| GND | GND |
| RST | **D9** |
| SDA (він же SS) | **D10** |
| MOSI | **D51** |
| MISO | **D50** |
| SCK | **D52** |
| IRQ | Не підключати |

> 💡 На Mega часто бачать пін **D53/SS** і думають, що RFID `SDA/SS` треба підключати туди. У цій прошивці `SS_PIN = 10`, тому `SDA/SS` підключай саме до **D10**. Якщо хочеш використати D53 — треба окремо змінити `SS_PIN` у прошивці.

> 💡 Дроти від MFRC522 короткі — використай перемички "мама-мама" і подовжуй до Arduino.

| VCC | 5V (червона шина) |
| GND | GND (синя шина) |
| TRIG | **D7** |
| ECHO | **D8** |

#### Датчик #2 — Слот 2

| HC-SR04 | Arduino |
|---------|---------|
| VCC | 5V |
| GND | GND |
| TRIG | **D4** |
| ECHO | **D6** |

#### Датчик #3 — Слот 3

| HC-SR04 | Arduino |
|---------|---------|
| VCC | 5V |
| GND | GND |
| TRIG | **A0** |
| ECHO | **A1** |

#### Датчик #4 — проїзд під воротами

Цей датчик треба ставити **прямо під воротами** так, щоб він "бачив" машину, коли та проїжджає.

| HC-SR04 | Arduino |
|---------|---------|

### Крок 5 — LCD дисплей I2C 16x2

LCD-модуль на задній стінці має маленьку плату-перехідник I2C з **4 пінами**: GND, VCC, SDA, SCL. Якщо у твого дисплея 16 пінів напряму — перевір що ти купив правильну версію (з I2C).

| LCD пін | Arduino |
|---------|---------|
| GND | GND (синя шина) |
| VCC | 5V (червона шина) |
| SDA | **D20 / SDA** |
| SCL | **D21 / SCL** |

> ⚠️ **Адреса LCD у прошивці — `0x27`** (рядок `LiquidCrystal_I2C lcd(0x27, 16, 2);`). Це найпоширеніша адреса для PCF8574. Деякі варіанти можуть мати `0x3F` — якщо після збірки дисплей світиться але нічого не показує, треба запустити **I2C Scanner** (стандартний приклад в Arduino IDE) і змінити адресу в коді.

> 💡 **На задній стінці LCD є синій потенціометр** — крути його викруткою, щоб налаштувати контрастність. На свіжо-підключеному дисплеї часто видно тільки сині прямокутники — це нормально, поверни потенціометр поки не з'являться чіткі літери.

> 💡 **Що буде показуватись на дисплеї:**
> - На старті: `Smart Parking` / `Controller Ready`
> - В режимі очікування: `Scan Card` / `Free:N/M`, де `N` — кількість вільних увімкнених місць, а `M` — кількість увімкнених місць
> - При піднесенні картки: `Access Granted`, `Blocked Card`, `Invalid Card`, `Access Locked`
> - При автовідкритті: `Auto Exit` / `Gate Open`

### Крок 6 — Адресні LED-стрічки маршрутів

У поточній прошивці використовується бібліотека **Adafruit NeoPixel**. Перед заливкою в Arduino IDE встанови її через Library Manager: `Adafruit NeoPixel`.

Кожне фізичне паркомісце має свою коротку LED-стрічку:

| LED-стрічка | До якого місця веде | DATA до Arduino | Живлення |
|-------------|---------------------|-----------------|----------|
| Strip #1 | **P1** | **D22** | +5V зовнішнього блока, GND спільний |
| Strip #2 | **P2** | **D23** | +5V зовнішнього блока, GND спільний |
| Strip #3 | **P3** | **D24** | +5V зовнішнього блока, GND спільний |

Підключення кожної стрічки однакове:

| Контакт LED-стрічки | Куди |
|---------------------|------|
| +5V | +5V зовнішнього блока живлення |
| GND | Спільна GND шина Arduino і зовнішнього блока |
| DIN / DATA | D22, D23 або D24 залежно від маршруту |

> ⚠️ Не живи LED-стрічки від 5V Arduino. Навіть короткі WS2812B можуть дати просадку живлення. Для макета краще взяти той самий зовнішній 5V блок, що планується для сервоприводу, але обов'язково зробити **спільну землю** з Arduino.

> 💡 Бажано поставити резистор **330–470 Ом** у розрив DATA кожної стрічки і конденсатор **1000 µF** між +5V та GND біля живлення стрічок.

Команди для перевірки через застосунок або Serial Monitor:

```text
PARKING ROUTE 1
PARKING ROUTE 2
PARKING ROUTE 3
PARKING ROUTE_CLEAR
```

## 7. Заливка прошивки

1. Встанови **Arduino IDE** з https://www.arduino.cc/en/software.
2. Запусти Arduino IDE → `File → Open` → вибери `SmartParkingSystemController.ino`.
3. У меню `Tools` встанови:
   - **Board**: `Arduino Mega or Mega 2560`
   - **Processor**: `ATmega2560`
   - **Port**: той, що з'явився після підключення USB (наприклад `COM3` на Windows).
4. Встанови потрібні бібліотеки через `Sketch → Include Library → Manage Libraries`:
   - **MFRC522** by GithubCommunity (для RFID)
   - **Adafruit NeoPixel** by Adafruit (для адресних LED-стрічок маршрутів)
   - **Servo** (зазвичай уже є)
   - HC-05 у поточній прошивці працює через `SoftwareSerial(2, 3)`
   - **LiquidCrystal I2C** by Frank de Brabander (**обов'язково**)
5. Переконайся, що HC-05 підключений саме до D2/D3, бо цього очікує поточна прошивка.
6. Натисни кнопку **Upload** (стрілочка вправо).
7. Чекай повідомлення `Done uploading`.
8. Якщо від'єднував HC-05 для діагностики — підключи назад D2/D3.

## 8. Перевірка що все працює

### 8.1. Перевірка LCD одразу після старту

Як тільки залив прошивку і Arduino перезавантажилось:

- На LCD має з'явитися: **`Smart Parking`** / **`Controller Ready`** на пару секунд.
- Потім перейде в режим очікування: **`Scan Card`** / **`Free:N/M`**, де `N` — кількість вільних увімкнених паркомісць, а `M` — кількість увімкнених паркомісць.

Якщо нічого не видно:
- **Світяться сині прямокутники, але немає літер** → крути потенціометр контрасту (синій гвинт ззаду LCD).
- **Дисплей повністю темний, не світиться** → перевір живлення (5V і GND).
- **Дисплей світиться але нічого не пише** → можливо інша I2C-адреса (`0x3F` замість `0x27`) — запусти приклад `I2C Scanner` і подивись яка адреса.

### 8.2. Важливо про Serial Monitor (USB)

У реальній прошивці команди застосунку йдуть **через Bluetooth HC-05 на D2/D3 (`SoftwareSerial`)**, а не через USB Serial Monitor.

USB Serial Monitor можна відкрити на **9600 baud** лише як допоміжну перевірку, що Arduino живиться і перезапускається. Надсилати `HELLO SPS`, `GET SNAPSHOT`, `GATE OPEN_TEMP` та інші команди треба через Bluetooth-підключення до HC-05.

> Для Wokwi-симуляції є окремий файл `SmartParkingSystemControllerWokwi.ino`: там команди спеціально приймаються через Serial Monitor, бо в симуляції немає реального HC-05.

### 8.3. Перевірка Bluetooth

1. На телефоні зайди в налаштування Bluetooth.
2. Знайди пристрій з ім'ям `HC-05` (PIN зазвичай `1234` або `0000`).
3. Спарся.
4. Відкрий будь-який термінал-додаток (наприклад **Serial Bluetooth Terminal** на Android).
5. Підключись до HC-05.
6. Надішли команду:
   ```
   HELLO SPS
   ```
7. У відповідь має прийти:
   ```
   |||HELLO_OK|device=SMART_PARKING|fw=2|slots=6|transport=HC05|||
   ```

### 8.4. Перевірка датчиків і воріт

Надішли по черзі:

| Команда | Що має статись |
|---------|---------------|
| `GET SNAPSHOT` | Прийде статус всіх слотів і воріт |
| `GET CONFIG` | Прийде поточна конфігурація |
| `GATE OPEN_TEMP` | Сервопривід поверне ворота → почекає 3 сек → закриє. На LCD на мить з'явиться повідомлення |
| `GATE FORCE_OPEN ON` | Ворота залишаться відкриті до `GATE FORCE_OPEN OFF` |
| `GATE LOCK ON` | Ворота заблокуються (RFID не зможе відкрити) |

### 8.5. Перевірка RFID

Піднеси картку до MFRC522. На LCD з'явиться один з варіантів (`Access Granted`, `Blocked Card`, `Invalid Card`), а в Bluetooth прийде:

- `EVENT|card=ALLOWED|uid=B041CE32` — ворота відкрились
- `EVENT|card=BLOCKED|uid=...` — картка у списку заблокованих
- `EVENT|card=INVALID|uid=...` — картка невідома

Щоб додати свою картку як дозволену, спочатку дізнайся її UID (піднеси — він прийде в `INVALID|uid=XXXXXXXX`), потім:

```
CARDS ALLOWED ADD XXXXXXXX
CONFIG SAVE
```

## 9. Часті проблеми та як їх ловити

| Симптом | Найімовірніша причина | Як виправити |
|---------|----------------------|--------------|
| Arduino перезавантажується щоразу при русі сервоприводу | Сервопривід просаджує живлення | Підключити сервопривід до зовнішнього блоку 5V 2A, землі з'єднати |
| MFRC522 не зчитує картки | Подано 5V замість 3.3V (модуль міг згоріти) АБО неправильні SPI-піни | Перевір живлення мультиметром (3.3V), перевір D10, D50, D51, D52 |
| Bluetooth не пара́ється | HC-05 не живиться, або плутанина TX↔RX | Подивись на світлодіод HC-05: блимає швидко = не спарений (норма), світить = спарений. Якщо не блимає — немає живлення |
| `HELLO SPS` не дає відповіді | Дроти TX і RX переплутані | Перевір: TX HC-05 → D2, RX HC-05 ← D3 через дільник |
| HC-SR04 показує -1 | Echo або Trig не підключений / поганий контакт | Перевір що обидва дроти у правильних пінах і breadboard добре тримає |
| Сервопривід "дзижчить" і не рухається | Слабке живлення | Зовнішній блок 5V |
| Прошивка не заливається (`avrdude: stk500_recv()`) | Не той порт у Tools→Port або не та плата в Tools→Board | Вибери `Arduino Mega or Mega 2560`, правильний COM-порт; за потреби від'єднай HC-05 на час перевірки |
| LCD: видно тільки сині прямокутники | Не налаштований контраст | Крути потенціометр на задній панелі LCD |
| LCD: дисплей світиться, але порожній | Інша I2C-адреса | Запусти `I2C Scanner` (приклад в Arduino IDE), глянь адресу. Якщо `0x3F` — поміняй у `.ino` рядок `LiquidCrystal_I2C lcd(0x27, 16, 2);` на `0x3F` |
| LCD не світиться взагалі | Немає живлення / переплутано VCC та GND | Перевір мультиметром що на VCC є 5V |
| Слоти 4-6 на старті вимкнені | Це норма | Прошивка тримає 6 логічних слотів, але фізично датчики є тільки на 1–3. За замовчуванням увімкнені P1-P3, а P4-P6 вимкнені. Їх можна увімкнути командами `PARKING ENABLE 4`, `PARKING ENABLE 5`, `PARKING ENABLE 6`, але без датчиків вони матимуть `distance_cm=-1` |

## Додаток B — Базові команди протоколу (шпаргалка)

Усі команди надсилаються через Bluetooth, у відповідь приходять фрейми обгорнуті в `|||...|||\n`.

| Команда | Дія |
|---------|-----|
| `HELLO SPS` | Хендшейк |
| `GET SNAPSHOT` | Поточний стан системи |
| `GET CONFIG` | Уся конфігурація (картки, тексти, налаштування) |
| `GET PROFILE` | Інфо про залізо |
| `GATE OPEN_TEMP` | Відкрити ворота на 3 сек |
| `GATE FORCE_OPEN ON` / `OFF` | Тримати відкритими |
| `GATE LOCK ON` / `OFF` | Заблокувати/розблокувати |
| `GATE CLOSE` | Закрити негайно |
| `CONFIG OPEN_ANGLE 90` | Поставити кут відкритих воріт |
| `CONFIG CLOSED_ANGLE 0` | Поставити кут закритих воріт |
| `CONFIG OPEN_DURATION_MS 5000` | Тримати відкритими 5 секунд після картки |
| `CONFIG THRESHOLD_CM 25` | Поріг "зайнятості" слоту |
| `CONFIG TELEMETRY_MS 500` | Інтервал телеметрії в мілісекундах |
| `CONFIG AUTO_EXIT_OPEN ON` / `OFF` | Увімкнути/вимкнути автовідкриття при наближенні машини |
| `CONFIG AUTO_CLOSE_AFTER_PASS ON` / `OFF` | Увімкнути/вимкнути автозакриття після проїзду |
| `CONFIG PASSAGE_THRESHOLD_CM 20` | Поріг датчика проїзду воріт |
| `CONFIG SAVE` | Записати в EEPROM (інакше після перезавантаження все скинеться!) |
| `CONFIG RESET` | Скинути все на дефолт |
| `PARKING ENABLE 1` / `PARKING DISABLE 1` | Увімкнути/вимкнути конкретний слот (1–6) |
| `CARDS ALLOWED ADD B041CE32` | Додати картку в дозволені |
| `CARDS BLOCKED ADD <UID>` | Додати в чорний список |
| `CARDS ALLOWED REMOVE <UID>` | Видалити |
| `CARDS BLOCKED REMOVE <UID>` | Видалити з чорного списку |
| `CARDS ALLOWED CLEAR` | Очистити список |
| `CARDS BLOCKED CLEAR` | Очистити чорний список |
| `DISPLAY FORCE ON` | Показати "FORCED TEXT" завжди |
| `DISPLAY FORCE OFF` | Вимкнути примусовий текст |
| `DISPLAY TEXT FORCED FORCED TEXT` | Змінити примусовий текст |
| `DISPLAY TEXT DEFAULT SMART PARKING` | Змінити idle-текст на дисплеї |
| `DISPLAY TEXT ALLOWED ACCESS GRANTED` | Змінити текст для дозволеної картки |
| `DISPLAY TEXT BLOCKED BLOCKED CARD` | Змінити текст для заблокованої картки |
| `DISPLAY TEXT INVALID INVALID CARD` | Змінити текст для невідомої картки |
| `DISPLAY TEXT LOCKED ACCESS LOCKED` | Змінити текст для заблокованих воріт |
