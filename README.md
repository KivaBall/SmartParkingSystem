# 🚗 Smart Parking System

Привіт! Це застосунок для **розумного паркінгу**, який допомагає оператору бачити, що відбувається на парковці, керувати воротами, дивитися події, працювати з RFID-картками та не тонути в купі сирих кнопок.

Система складається з **MAUI-застосунку**, **Arduino-контролера** і додаткового **Telegram-бота**, який може надсилати сповіщення про стан паркінгу.

## ✨ Що ця система вміє

- 🅿️ показувати, які місця **вільні**, **зайняті**, **вимкнені** або знаходяться на іншому поверсі
- 🏢 працювати з **двоповерховою картою паркінгу**
- 🚧 керувати **воротами**: відкрити, закрити, заблокувати, тримати відкритими
- 📡 автоматично відкривати/закривати ворота через **датчик проїзду**
- 💳 додавати, видаляти та блокувати **RFID-картки**
- 📟 змінювати тексти на **LCD-дисплеї Arduino**
- 📷 показувати **live-камеру біля воріт** і зберігати знімки після відкриття
- 🧾 зберігати локальні події, знімки камери, мову та backend-налаштування
- 🤖 надсилати сповіщення в **Telegram**, включно зі знімками камери

## 📱 Основні вкладки застосунку

- `Connection` — підключення до Arduino-контролера та перевірка, що це саме наш пристрій.
- `Dashboard` — короткий огляд стану паркінгу, воріт, карток і контролера.
- `Parking` — карта паркомісць, вибір поверху, вибране місце та режим редагування.
- `Gate` — керування воротами та live-камера біля в'їзду.
- `Activity` — події системи та збережені фото з камери.
- `Admin` — налаштування контролера, автоматика воріт, камера, картки, слоти й системні параметри.
- `Settings` — мова, backend URL і перемикач зв'язку з backend.

## 🏗️ Що де лежить

- `SmartParkingSystem.Maui` — головний MAUI Blazor Hybrid застосунок для Windows та Android.
- `SmartParkingSystem.Domain` — спільні моделі, які використовують MAUI та Telegram backend.
- `SmartParkingSystem.TelegramBot` — ASP.NET Core backend + Telegram bot + SignalR + SQLite.
- `arduino/` — прошивки, Wokwi/Demo варіанти та інструкції по підключенню заліза.
- `docs/` — архітектурні правила, опис структури, Tailwind whitelist та загальна документація.
- `assets/` — картинки й графічні матеріали.

## 🔌 Arduino Hardware

Основна прошивка зараз тут:

- `arduino/SmartParkingSystemController.ino`

Інструкції для підключення:

- `arduino/HARDWARE_SETUP_ArduinoUno.md` — схема для Arduino Uno.
- `arduino/HARDWARE_SETUP_ArduinoMega.md` — схема для Arduino Mega 2560.

Важливий нюанс для Mega: поточна прошивка все ще використовує `SoftwareSerial btSerial(2, 3)`, тому HC-05 підключається до `D2/D3` навіть на Mega. Для Mega відрізняються саме SPI/I2C піни: RFID використовує `D50-D52`, LCD використовує `D20/D21`.

## 🅿️ Поточна логіка паркомісць

- Прошивка тримає **6 логічних паркомісць**.
- Фізично датчики зараз підключені для **P1-P3**.
- За замовчуванням **P1-P3 увімкнені**, а **P4-P6 вимкнені**.
- P4-P6 можна увімкнути командами, але без датчиків вони матимуть `distance_cm=-1`.

## 🤖 Telegram Bot

Telegram-бот живе в окремому проєкті `SmartParkingSystem.TelegramBot`.

Він:

- приймає snapshot-и з MAUI через SignalR;
- зберігає стан у SQLite;
- має окремі налаштування для кожного Telegram-чату;
- показує інтерактивне меню через `/start`;
- може надсилати сповіщення про ворота, паркінг, адмін-зміни та камеру;
- може надсилати фото, які MAUI зробив з камери.

Секрети типу Telegram bot token **не треба комітити**. Їх потрібно задавати через локальну конфігурацію або environment variables.

## 🧠 Що зберігається локально

MAUI-застосунок зберігає на пристрої:

- вибрану мову;
- backend URL;
- чи увімкнений backend sync;
- історію подій;
- знімки камери.

Telegram backend окремо зберігає свій стан у SQLite.

## 🤖 Команди до Arduino

Нижче коротка шпаргалка команд, які використовуються для роботи з контролером. Детальніше вони також описані в hardware setup файлах.

| Команда | Що робить |
|---|---|
| `HELLO SPS` | перевіряє, що це саме наш контролер |
| `GET PROFILE` | просить профіль пристрою |
| `GET CONFIG` | просить поточну конфігурацію |
| `GET SNAPSHOT` | просить поточний знімок стану |
| `GATE FORCE_OPEN ON` / `OFF` | вмикає/вимикає примусово відкриті ворота |
| `GATE OPEN_TEMP` | тимчасово відкриває ворота |
| `GATE CLOSE` | закриває ворота |
| `GATE LOCK ON` / `OFF` | блокує/розблоковує ворота |
| `CONFIG SAVE` | зберігає конфігурацію в EEPROM |
| `CONFIG RESET` | скидає конфігурацію до дефолту |
| `CONFIG OPEN_ANGLE <value>` | ставить кут відкриття воріт |
| `CONFIG CLOSED_ANGLE <value>` | ставить кут закриття воріт |
| `CONFIG OPEN_DURATION_MS <value>` | ставить тривалість тимчасового відкриття |
| `CONFIG THRESHOLD_CM <value>` | ставить поріг зайнятості паркомісць |
| `CONFIG TELEMETRY_MS <value>` | ставить інтервал телеметрії |
| `CONFIG AUTO_EXIT_OPEN ON` / `OFF` | вмикає/вимикає автовідкриття для виїзду |
| `CONFIG AUTO_CLOSE_AFTER_PASS ON` / `OFF` | вмикає/вимикає автозакриття після проїзду |
| `CONFIG PASSAGE_THRESHOLD_CM <value>` | ставить поріг датчика проїзду воріт |
| `PARKING ENABLE <slot>` | вмикає паркомісце |
| `PARKING DISABLE <slot>` | вимикає паркомісце |
| `CARDS ALLOWED ADD <uid>` | додає дозволену картку |
| `CARDS ALLOWED REMOVE <uid>` | видаляє дозволену картку |
| `CARDS ALLOWED CLEAR` | очищає список дозволених карток |
| `CARDS BLOCKED ADD <uid>` | додає заблоковану картку |
| `CARDS BLOCKED REMOVE <uid>` | видаляє заблоковану картку |
| `CARDS BLOCKED CLEAR` | очищає список заблокованих карток |
| `DISPLAY FORCE ON` / `OFF` | вмикає/вимикає примусовий текст дисплея |
| `DISPLAY TEXT FORCED <text>` | задає примусовий текст |
| `DISPLAY TEXT DEFAULT <text>` | задає стандартний текст |
| `DISPLAY TEXT ALLOWED <text>` | задає текст для дозволеної картки |
| `DISPLAY TEXT BLOCKED <text>` | задає текст для заблокованої картки |
| `DISPLAY TEXT INVALID <text>` | задає текст для невідомої картки |
| `DISPLAY TEXT LOCKED <text>` | задає текст для заблокованих воріт |
